HTML

  • HTML is made up of elements that work together to provide content and the basic formwork for all pages on the web.
  • Content Elemnts: Text, Image, Header, List
  • Element types used to divide and describe a page: Meta-related elements, Content Division elements, Semantic elements.
  • The head element contains information about the webpage to help the browser render the page correctly. Not visible to users.
  • The body element represents the visible content shown to the user.
  • The footer element usually contains the author, copyright, contact, sitemap, and navigation.
  • HTML Attribute: defines additional information about an element. HTML attributes are alwyays defined in the beginning element, never the closing element.
  • The "style" element defines the CSS styles associated with a website.
  • The "title" element defines the title for a webpage.
  • The "base" element defines the base URL for a webpage.
  • The "link" element connects an external resource to the HTML document.
  • The "meta" element defines metadata such as the character set, description, keywords, author, and viewport.
  • The "script" element embeds client-side scripts in the HTML document.
  • The "charset" elements sets our character encoding.
  • The "http-equiv" element sets an HTTP header for content.
  • The "viewport" element sets the default view for our webpage.
  • Header elements range from "h1" to "h6" and should be used in order. The "h1" header should only be used once on each page.
  • Other common Body elements include: "p" for paragraph, "ul" for an unordered, or bulletpointed list, "ol" for an ordered list, and "li" for list items.
  • Image elements, "img", are used to display images and contain various attributes such as "src" defining the location of the image.

CSS

  • Cascaiding Style Sheets (CSS) defines the colors, aesthetics, visual layout, and organization.
  • Three ways to style a webpage: inline CSS, internal CSS style sheet, external CSS style sheet.
  • Inline CSS - code added to the HTML file, applying to a specific HTML tag to style a particular page element only.
  • Example of Inline CSS: Using the "style" attribute to change the color of a single "h1" heading - ie. "h1 style="color:red;""
  • Internal Style Sheet: embeds an entire stylesheet directly in the HTML file using a "style" element in the "head" element.
  • Internal Style Sheet: The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code. A single change to the CSS rule will apply to all tagged page elements.
  • External Style Sheet: The most common way to style a page. Preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • External Style Sheet: Link the CSS file to the HTML file using the "link" element placed in the HTML's "head".
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class Attributes: Assign styling to specific elements by using Class Attributes. To designate a selector as a class selector use a ".".

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main: moves working branch to main.
  • git add -A: adds files from the working branch to the stage area.
  • git commit -m "commit message": commits files from your local environment. Like a local save.
  • git pull origin main: pulls the current updated main files from GitHub to local environment.
  • git push origin "branch/name": pushes the branch from local environment to GitHub.
  • The GitFlow:
    1. git add -A
    2. git commit -m "commit message"
    3. git pull origin main
    4. git push origin (branch/name)
    5. Create Pull Request in Github
    6. Merge the Pull Request in Github
    7. git checkout main
    8. git pull origin main

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Conditional statements, like IF statements, can be used to interrupt control flow.
  • Variables that store more than one value are called arrays.
  • Arrays can store values of varying data types.
  • Array values are tracked by indexing using brackets and starting with the value 0.
  • A FOR loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • FOR loops contains 3 important statements: starting point for the loop, condition, iteration mechanism.
  • A Function is a set of instructions that tells the computer how to perform a certain task that must be "called" in order to execute.
  • First you define the Function by using the FUNCTION keyword followed by the unique name of the Function and parentheses, then with executable code within curly brackets.
  • To Call the function, simply type the name of the function where you want the function executed with your code.
